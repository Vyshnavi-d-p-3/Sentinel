# Eval fixtures

Pull-request JSON files drive the P/R/F1 harness (`eval/scripts/eval_runner.py`).

## What’s in this directory

- **`synth_pr_001.json` … `synth_pr_100.json`** — generated by
  `python eval/scripts/generate_synthetic_fixtures.py` so ground truth matches
  **`LLM_MOCK_MODE`**: one comment at the diff’s first `+++ b/` path and `@@ +N`
  line, with **category / severity** from
  `backend/app/services/mock_label_anchor.py` (spreads labels across security,
  bug, performance, style, suggestion and low/medium/high). This keeps **CI**
  regression metrics stable. Regenerate with:

  `python eval/scripts/generate_synthetic_fixtures.py --out eval/fixtures --count 100`
- **`legacy/`** — hand-crafted examples (not loaded by the runner; see
  `legacy/README.md`).

## Real hand labels

For **non-synthetic** benchmarks, add or replace fixtures using
`eval/scripts/labeling_rubric.md`, then re-run the runner and **refresh**
`eval/baselines/baseline.json` (see `eval_runner.py` and CI).

## Adding a fixture

1. Copy a template from [`legacy/`](legacy/) (e.g. `example_pr_001.json`) or from any `synth_pr_*.json` for structure.  
2. Use real unified diff text; avoid synthetic empty hunks.  
3. Re-run:  
   `python eval/scripts/eval_runner.py --fixtures eval/fixtures/ --output eval/results.json`  
4. Update `dataset_version` in `eval_runs` / baseline when the set materially changes.

See also [`../README.md`](../README.md) for the full eval layout and [`../../docs/PUBLISHING_AND_BENCHMARK.md`](../../docs/PUBLISHING_AND_BENCHMARK.md) for publishing methodology.
