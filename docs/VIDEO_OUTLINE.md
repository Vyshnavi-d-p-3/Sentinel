1. **Title card (10s)** — “Sentinel: AI code review with reproducible eval”
2. **Problem (20s)** — Most “AI review” is a thin wrapper; measurement matters.
3. **Architecture (40s)** — One slide: GitHub → webhook → retrieval + LLM → Check Run; show **hybrid** retrieval in one line.
4. **Live: GitHub (45s)** — Open a test PR (or smee) → show review / Check Run appearance.
5. **Dashboard (50s)** — `Reviews` row → `Eval` page with F1 by category; mention **regression gate** in CI.
6. **Cost / settings (20s)** — `Costs` or token budgets, mock vs live.
7. **Close (15s)** — Repo link, “eval harness is the differentiator,” CTA to star or read the blog.

**Recording tips:** 1080p, system audio off, use one browser window, rehearse the click path once.
